Projects

Two-Cloud Private Read Alignment to a Public Reference Genome

The large-scale nature of human genome necessitates outsourcing of genomic computations to public clouds. However, this raises some serious privacy concerns. We designed and developed a two-cloud private read alignment algorithm using the Burrows-Wheeler Transform and the FM-Index. Our paper is conditionally accepted to Proceedings of Privacy Enhancing Technologies Symposium 2023.

Graph Sktches and Embeddings: A study of their Applications on Graph Databases (Master's Thesis)

Large-scale graph processing has evolved to be an important research area, thanks to the proliferation of on-line social networks and growing uses of linked data. An important practical consideration while processing large volumes of data is creating summaries of the data, since it is usually impractical to perform operations on the entire dataset. In my thesis, I implemented five sparsification techniques and four graph embedding techniques on LDBC SNB social networks data of three different scale factors ran cypher queries for connected components, community detection, page rank, partition size and betweenness centrality on data before and after sparsification and provided a performance study of these techniques, ran cypher query cosine similarity on embedded and non-embedded data using different techniques in neo4j database, implemented pairwise cosine similarity algorithm to compare and evaluate the results, provided visualization for sparsified, unsparsified, embedded and unembedded graph data to deal with two most important challenges in large-scale graph processing: scalability and visualization.

Evaluation of High-Performance Application-level Caching of Generic Graph Structured Data

As a team, we researched different hashing techniques and implemented simple graph algorithms (adjacency lists, shortest path, and the minimum number of hops) on SNAP datasets. We implemented five hashing techniques on this graph data and made a comparison by loading the data in neo4j graph database.

Supporting non-relational models with flexible schema storage

A python based scientific project that that deals with the comparison of performances of NoSQL data stores (Key-value stores and document stores) with relational data models in terms of use of different operators (get and count operator for key-value stores and document stores respectively). For this we have implemented two operators: get and count on two non-relational models: key-value stores and document stores respectively and compared the performance with the relational model using two TPC-H datasets of different selectivity factors and evaluated the results. In addition to the evaluation of KVS and DS queries, we also explored the tile-based architectures and provided a theoretical discussion on how the mappings from different NoSQL data stores and relational model to tile-based architectures might work.
